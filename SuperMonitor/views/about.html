Introduction

Provide an overview and introduction to the monitoring app.

Workspaces

A workspace is a logical grouping of monitoring configurations and data. It provides an isolated environment for managing and organizing monitoring tasks. There are two types of workspaces:

API Workspace

API workspaces allow you to monitor API endpoints. You can configure the endpoint details such as URL, request headers, and polling rate.

SQL Workspace

SQL workspaces enable monitoring of SQL databases. You can specify the database connection details such as host, port, user, password, and database name. Additionally, you can define custom SQL queries to fetch data from the database.

Connectivity

The monitoring app supports two types of connectivity:

API Connectivity

With API connectivity, you can monitor external API endpoints. Configure the endpoint details such as URL, request method, request headers, and other parameters.

SQL Connectivity

SQL connectivity enables monitoring of SQL databases. Specify the database connection details such as host, port, user, password, and database name to establish a connection.

Dashboards

Dashboards provide visual representations of the monitored data. You can create custom dashboards to visualize and analyze the collected metrics. Dashboards can include charts, graphs, and other visualization components.

Hyperparameters

The monitoring app supports various hyperparameters for fine-tuning the monitoring process:

Scheduled Polling Rate

You can set the frequency at which the monitoring app polls the endpoints or databases for data updates.

Request Headers

For API connectivity, you can specify custom request headers to be included in the API requests.

SQL Queries

In SQL workspaces, you can define custom SQL queries to fetch specific data from the connected databases.
